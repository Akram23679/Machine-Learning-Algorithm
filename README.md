# Machine-Learning-Algorithm

A structured, growing collection of my machine learning notes — organized by algorithm family, with explanations and code for each. I built this as a personal reference while learning, and as a place to keep clean, worked examples I can come back to.

> 📌 If you're browsing this from my profile: each folder holds notes and implementations for a specific group of algorithms. Start with whichever family interests you.

---

## 📂 Repository Structure

```
Machine-Learning-Algorithm/
│
├── Supervised Learning/
│   ├── Classification/     # Predicting categories / labels
│   ├── Regression/         # Predicting continuous values
│   └── Ensemble/           # Combining models for better performance
│
└── Unsupervised Learning/
    └── Clustering/         # Finding groups in unlabeled data
```

---

## 🧠 What's Inside

### Supervised Learning
Algorithms that learn from **labeled** data — each example comes with the correct answer.

| Folder | What it covers | Example algorithms |
|---|---|---|
| **Classification** | Predicting a category or class | Logistic Regression, K-Nearest Neighbors, Decision Trees, Naive Bayes, SVM |
| **Regression** | Predicting a continuous number | Linear Regression, Polynomial Regression, Ridge / Lasso |
| **Ensemble** | Combining many models into a stronger one | Random Forest, Gradient Boosting, AdaBoost, XGBoost |

### Unsupervised Learning
Algorithms that find structure in **unlabeled** data — no correct answers provided.

| Folder | What it covers | Example algorithms |
|---|---|---|
| **Clustering** | Grouping similar data points together | K-Means, Hierarchical Clustering, DBSCAN |

---

## 🎯 Purpose

These are study notes with runnable examples — the goal is to understand *how* each algorithm works, not just call it. For each algorithm I try to include:

- A short plain-language explanation of the idea
- When to use it (and when not to)
- A worked example with code
- Key parameters and how they affect the result
